## SECTION 3. Phase loop trace

Fresh invocation trace:

1. `AGENTS.md` routes a default first user message to `prompts/prove.md` with the message as `$1`. Artifact target: new `problems/<slug>/`. Next prompt: `prompts/prove.md`.
2. `prompts/prove.md` reads core files, creates slug, copies `templates/problem_skeleton/`, writes `problem.md`, writes `problems/.active`, initializes `session_state.md` and `claim_ledger.md`. Next prompt: `prompts/00-intake.md`.
3. `prompts/00-intake.md` writes `intake.md`, initializes ledger/state if absent, and may preload corpus citations. Next prompt: `prompts/checkpoint.md`.
4. `prompts/checkpoint.md` updates ledger, work journal, session state, and emits MSP R5. Next prompt per `prove.md`: `prompts/05-review.md` reviewing intake.
5. `prompts/05-review.md` writes reviewer artifacts under `reviews/` and `review_consolidated.md`, then invokes checkpoint. Next prompt: `prompts/checkpoint.md`, then `prompts/01-survey.md`.
6. `prompts/01-survey.md` writes `survey.md` with literature tree, state of art, and terrain map. Next prompt: checkpoint, then Phase 5 review of survey.
7. `prompts/02-perspective.md` writes `perspectives.md`, using registered specialists or `specialist-factory.md`. Next prompt: checkpoint, then Phase 5 review of perspectives.
8. `prompts/03-hypothesize.md` writes `hypotheses_live.md`, sanity-check log, and v4 direct-attack/retreat-trigger state. Next prompt: checkpoint, then Phase 5 review of hypotheses.
9. `prompts/04-develop.md` writes `proofs/<id>.fml` and/or `failures/<id>.md`, ledger rows, and v4 STUCK-STATE/NEXT-SESSION ATTACK PLAN if stuck. Next prompt: checkpoint, then Phase 5 review of development.
10. `prompts/06-consolidate.md` writes `result.md`, `result.fml`, `executive_summary.md`, optional corpus entries, and updates `corpus/INDEX.md` if promoted. Next prompt: checkpoint, then Phase 5 review of consolidation.
11. `prompts/halt.md` writes `resume_brief.md`, updates session state, and emits wind-down digest.

Precondition/failure section check: none of the seven phase prompts has a literal `[PRECONDITIONS]` section. They use `[INPUTS]` and `[FAILURE]` instead. Failure sections correctly point to the prior artifact for Phase 1 (`intake.md`), Phase 2 (`survey.md`), Phase 3 (`perspectives.md`), Phase 4 (`hypotheses_live.md`), and generic review/consolidation blockers for Phases 5/6. Phase 0 correctly fails on missing `problem.md`.

Blocking trace issue: `prompts/05-review.md` v3 invariant says 05-review MUST be invoked AFTER `04-develop` artifacts exist. This contradicts `prompts/prove.md`, `goal.md`, and the design intent requiring Phase 5 review after every non-review phase, including intake/survey/perspective/hypothesize/consolidate.

Major trace issue: `prompts/05-review.md` input list requires `survey.md`, but the phase loop invokes review immediately after Phase 0 intake, before `survey.md` exists.

## SECTION 10. Discrepancies and recommendations

BLOCKING: `prompts/05-review.md` says 05-review MUST be invoked after `04-develop` artifacts exist, contradicting the phase loop and design intent that Phase 5 reviews each non-review phase. Recommendation: scope that invariant specifically to review of 04-develop artifacts, or replace it with "05-review MUST be invoked after the artifact it reviews exists."

BLOCKING: `prompts/05-review.md` mandates Persona D, but its procedure invokes only A/B/C, and `prompts/subagents/reviewer.md` rejects unknown personas outside `{A,B,C}`. Recommendation: update both `05-review.md` procedure and `reviewer.md` contract to include D with phase gating.

MAJOR: `prompts/05-review.md` input list requires `survey.md`, which is unavailable for the immediate review of Phase 0 intake. Recommendation: make `survey.md` optional before Phase 1 or split review inputs by reviewed phase.

MAJOR: No phase prompt has a `[PRECONDITIONS]` section, despite the stated audit/design expectation. Recommendation: add explicit precondition sections to `00-intake.md` through `06-consolidate.md` and ensure they name prior artifacts.

MAJOR: `corpus/theorems/` is missing while `06-consolidate.md` requires writing self-contained entries there. Recommendation: add `corpus/theorems/.gitkeep`.

MAJOR: `AGENTS.md` routes `muse` and `consult <specialist>` to "corresponding prompt", but no top-level prompts exist. Recommendation: add prompt wrappers or rewrite routes to subagent paths.

MAJOR: Several subagents are orphaned from the phase graph (`prover`, `surveyor`, `sanity-checker`, `skill-author`; `muser` weakly routed). Recommendation: wire expected subagents into `01-survey.md`, `03-hypothesize.md`, `04-develop.md`, and AGENTS routes where intended.

MAJOR: `goal.md` requires per-unit `context_manifest.md`, but there is no template or checkpoint rule creating it. Recommendation: add a template and checkpoint/prove references.

MAJOR: `prompts/resume.md` can clear `halt_flag=true`, while v4 closed states also set `halt_flag=true`. Recommendation: guard this step so only resume-eligible states clear halt flags.

MINOR: `notes/formalization/` is referenced but not present in the problem skeleton. Recommendation: add `templates/problem_skeleton/notes/formalization/.gitkeep`.

MINOR: `verbose`/`quiet` toggling has no explicit session_state field. Recommendation: add a field such as `msp_narration: quiet | verbose`.

OBSERVATION: v4 commit `2f0d0a6` modified 10 files, not 9. This matches the explicit append sections supplied for v4, including both session state templates.

OBSERVATION: The v4 ambition/persistence concept is coherent at the policy level: direct attack first, earned retreat only after documented stuck state, meaningfulness gate before corpus promotion, and BLOCKED-ITERATE resume loop. The main issues are integration/wiring conflicts rather than the ambition model itself.

Executive summary: The v4-patched repo is directionally coherent with the stated design intent, but not yet mechanically coherent enough to trust the full loop without further patching. The top three highest-severity items are: (1) the Phase 5 invariant that restricts review to after 04-develop conflicts with the required review-after-each-phase loop; (2) Persona D is mandated but unsupported by the review procedure and reviewer subagent contract; (3) Phase 5 requires `survey.md` even when reviewing Phase 0 before survey exists. Address those first, then add missing precondition sections and corpus/context directories.
